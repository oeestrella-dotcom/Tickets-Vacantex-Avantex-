Usuarios del sistema

Sobre la seguridad: las contraseñas ya no se guardan en texto — las administra Firebase Authentication (el mismo sistema de autenticación de Google), cifradas y fuera del alcance de cualquiera, incluido tú como administrador. Al "resetear" la contraseña de alguien, en vez de generarle una nueva a la vista, el sistema le envía un correo de Firebase para que la persona misma defina una nueva — es el estándar de la industria. Aun así, este sigue siendo un control de acceso para uso interno, no una plataforma con auditorías o SSO de nivel empresarial.

Da de alta a cada supervisor con su correo corporativo como usuario. El sistema genera la contraseña automáticamente y te la muestra una sola vez para que se la compartas.

Alertas por correo

Este sistema no puede enviar correos de forma automática y silenciosa desde el navegador (no tiene un servidor detrás). Lo que sí hace: te avisa en el Dashboard qué vacantes necesitan aviso y te prepara el correo listo — solo das clic en "Enviar" y se abre en tu programa de correo (Outlook, Gmail, etc.) ya redactado y dirigido al reclutador. Define aquí a partir de cuántos días quieres que aparezca esa alerta.

Este panel te dice qué avisar y te deja el correo listo con un clic — pero que salga solo, todos los días, sin que nadie abra la app, requeriría una tarea programada (Cloud Functions con temporizador), que en Firebase ya pide activar el plan de pago por uso (aunque el uso normal de esto se quede en $0). Si más adelante te interesa, te ayudo a dejarlo armado.

Acerca de este hospedaje

Cómo está armado este sistema para que puedas mantenerlo

Este archivo está hospedado como página estática (por ejemplo en GitHub Pages) y usa Firebase (plan gratuito Spark) como base de datos y autenticación, para que los datos y usuarios se compartan entre cualquier computadora o celular con solo el enlace, usuario y contraseña. Revisa el archivo README.md del repositorio para el paso a paso de configuración inicial.
